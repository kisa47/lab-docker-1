## Проведение лабораторной работы

### Подготовка образа контейнера с приложением

Необходимо подготовить и собрать образ контейнера с приложением написанным на языке **Python**, исходные коды которого размещены в директории ```src```.

Для создания образа Docker контейнере необходимо создать файл **Dockerfile** в корне лабораторной работы и заполнить его согласно следующим требованиям:

- Для работы приложения необходим интерпритатор **python** версии **3.12** [Подсказка](#подсказка-1)
- Рабочая директория **/lab-docker-1**
- Для работы приложения необходимы библиотеки. При разработке ПО необходимые библиотеки (зависимости) которые указываются с определёнными версиями в файле [requirements.txt](./src/requirements.txt). Установка **python** библиотек указанных в файле выполняется командой: ```pip install --no-cache-dir -r requirements.txt```
- Необходимо скопировать код приложения из директории *./src* в рабочую директорию образа.
- Так как по умолчанию отладку мы не будет включать в контейнере, необходимо задать значение переменной **FLASK_ENV** равное ```production```
- Приложение запускается командой ```python3 app.py```
- Собранный образ должен иметь имя ```lab-docker-1``` [Подсказка](#подсказка-2)

После создания и заполнения **Dockerfile** необходимо выполнить сборку образа. [Подсказка](#подсказка-3).

Если образ собрался без ошибок, то необходимо записать команду сборки в разделе[Полученные результаты](#полученные-результаты).

### Запуск приложения в контейнере

Приложение написано адаптированным для запуска в контейнере, поэтому все необходимые настройки задаются через **переменные окружения**. Соответственно при запуске такого приложения нет необходимости создавать какой либо файл конфигурации и достаточно просто задать переменные окружения. 

Используемые переменные окружения:

- **USER_NAME** - ваша фамилия и имя. Например ```Сурцов В.```
- **USER_GROUP** - ваша группа
- **LISTEN_PORT** - порт, который прослушивает приложение (по умолчанию **8080**).
- **SAVED_FILE** - путь до файла, в который будут сохранятся данные.

Вам необходимо выполнить запуск контейнера таким образом выполнив следующие требования:
- Задать Ваше имя и фамилию с помощью переменной окружения **USER_NAME** [Подсказка](#подсказка-4)
- Задать вашу группу с помощью переменной окружения **USER_GROUP**
- Порт хоста, которые будут перенаправляться в контейнер должен быть **8080**
- Порт, который слушает приложение в контейнере должен быть также **8080** [Подсказка](#подсказка-5)
- Необходимо выключить автоматическое удаление контейнера после его выключения [Подсказка](#подсказка-6)
- Задать имя контейнеру **lab-docker-1-1** [Подсказка](#подсказка-7)

После запуска контейнера проверить работу приложения, для этого необходимо в браузере перейти по [адрессу](http://localhost:8080).

На странице приложения должны корректно отображаться заданные вами параметры.

В случае, если параметры отображаются некорректно нужно остановить работу приложения (контейнера) сочетанием клавиш клавиатуры **Ctrl+C** и выполнить запуск повторно с исправленной командой запуска.

Если приложение работает, то необходимо записать команду запуска в разделе[Полученные результаты](#полученные-результаты).

Далее в WEB интерфейсе приложения необходимо ввести в поле следующий текст:
```
Проверка работы приложения.
```
И нажать кнопку **Сохранить**. После чего в графе **Содержимое файла** должен быть указан введённый вами текст.

Если текст указан корректно нужно остановить работу приложения (контейнера) сочетанием клавиш клавиатуры **Ctrl+C** и выполнить запуск повторно с использованием той же команды.

В в разделе [Полученные результаты](#полученные-результаты) необходимо записать, сохранилось ли содержимое файла после перезапуска контейнера **lab-docker-1-1** и ответить почему на ваш взгляд это произошло?

### Запуск контейнеров с сохранением в файловой системе хоста

Создадим директорию для тома **uploads**, где будем хранить изменяемые файлы контейнера командой:
```bash
mkdir uploads
```

Далее произведем запуск двух контейнеров с прошлыми настройками, но с дополнительными требованиями:
- Контейнеры должны работать в фоновом режиме [Подсказка](#подсказка-8)
- Имена контейнеров должны быть **lab-docker-1-2** и **lab-docker-1-3**.
- Порты для перенаправления на контейнеры **lab-docker-1-2** и **lab-docker-1-3** должны быть **8080** и **8081** соответственно.
- К каждому контейнеру должена быть примонтирована директория хоста **uploads** в директорию **/app/uploads** контейнера. [Подсказка](#подсказка-9)

Проверить работу двух экземпляров приложения через браузер. Ссылкой для получения доступа будет ```http://localhost:8080```, где **8080** - порт хоста, который перенаправляется в контейнер. Также следует проверить работает ли сохранение файла в файловой системе хоста, для этого в WEB интерфейсе одного из экземпляров приложения введем в поле для ввода ```test``` и нажмём кнопку **Сохранить**. Если всё работает правильно при обновлении WEB страницы второго экземпляра приложения содержимое файла должно обновиться на ```test```, а также в директории **uploads** должен появиться файл с содержанием ```test```.


Если работает 2 экземпляра приложения, записать команды запуска контейнеров в 
[Полученные результаты](#полученные-результаты).

Далее перейдем в браузере к приложению, работающему в контейнере **lab-docker-1-2** по [адрессу](http://localhost:8080) и введем в поле для ввода **lab-docker-1-2** нажмём сохранить.

Далее перейдем в браузере к приложению, работающему в контейнере **lab-docker-1-3** по [адрессу](http://localhost:8081) и введем в поле для ввода **lab-docker-1-3** нажмём сохранить.

Далее перезапустим контейнеры командами:
```bash
docker restart lab-docker-1-2
docker restart lab-docker-1-3
```

И проверим в Web интерфейсе обоих экземпляров приложения содержимое файла **saved_file.txt**. Результат проверки запишем в 
[Полученные результаты](#полученные-результаты).

Также необходимо ответить на вопрос "Почему на ваш взгляд после перезапуска контейнеров **lab-docker-1-2** и **lab-docker-1-3** содержимое именно такое?" в разделе [Полученные результаты](#полученные-результаты).

Далее получим список запущенных контейнеров командой:
```bash
docker ps
```

Далее получим версию установленного интерпритатора **python** в контейнере **lab-docker-1-2** и запишем в разделе [Полученные результаты](#полученные-результаты):
```bash
docker exec lab-docker-1-2 python3 --version
```

Получим список установленных **python** библиотек в контейнере **lab-docker-1-2** и запишем в разделе [Полученные результаты](#полученные-результаты):

```bash
docker exec lab-docker-1-2 pip list
```

После чего выполним команды на хосте для получения версии интерпритатора и установленных **python** библиотек на хосте и запишем в разделе [Полученные результаты]:
```bash
python3 --version
pip list
```

После завершения выполнения работы необходимо остановить контейнеры командами:
```bash
docker restart lab-docker-1-2
docker restart lab-docker-1-3
```

## Подсказка 2
При сборке образа имя можно задать через параметр **-t/--tag**, например:
```bash
docker build . -t name
```
## Подсказка 3
При сборке образа необходимо задать правильно контекст сборки. Контекст зависит от путей, которые использовались при заполнении **Dockerfile** и является корнем контекста.

**Dockerfile** по умолчанию берется из корня контекста сборки. Если файл

Если в **Dockerfile** пути указаны относительно директории с лабораторной работой, то контекст нужно указывать директорию Например если в **Dockerfile** указано ```COPY src/requirements.txt .```, то в качестве контекста стоит указать корень директорию проекта:
```bash
docker build .
```
Если в **Dockerfile** пути указаны относительно директории ```src```, например ```COPY requirements.txt .```, то в качестве контекста стоит указать директорию ```src```:
```bash
docker build ./src
```

## Подсказка 4
При запуске контейнера переменные окружения задаются с помощью параметра **-e/--env**. Например:
```bash
docker run -e FOO='B A R' hello-world
```
При этом, если переменные содержат спцесимволы (например,пробелы) то значение переменное нужно брать в ковычки:
```bash
docker run -e FOO='B A R' hello-world
```

## Подсказка 5
Перенаправление портов задается с помощью параметра **-p/--publish list**. Например, для перенаправления запросов с порта **80** хоста на порт контейнера **999** необходимо выполнить:
```bash
docker run -p 80:999 hello-world
```

## Подсказка 6
Чтобы контейнер автоматически удалился после его выключения используется параметр **--rm**, например:
```bash
docker run --rm hello-world
```

## Подсказка 7
Чтобы задать имя контейнеру используется параметр  **--name**, например:
```bash
docker run --name my-first-container hello-world
```

## Подсказка 8
Для запуска контейнера в фоновом режиме (detached) используется параметр  **-d/--detach**, например:
```bash
docker run -d hello-world
```

## Подсказка 9
Для монтирования директории хоста к контейнеру исползьуется параметр **-v/--volume**, например:
```bash
docker run -v path/to/host/dir:/path/to/container/dir hello-world
```
Относительный путь директории хоста указывается относительно текущего положения.
Относительный путь директории контейнера указывается относительно рабочей директории (**WORKDIR**) контейнера.
